# 🎮 Discord 音樂問答游戲 - 開發計畫

## 📋 項目概述

為 Aoede Discord 音樂機器人添加互動問答游戲功能，讓觀眾可以猜測正在播放的歌曲信息。

### 核心功能
- **Slash Commands**：觸發游戲開始 (`/quiz` 或 `/猜歌`)
- **Message Components**：按鈕式互動選擇答案
- **多種題型**：猜動漫、猜歌手、猜歌名
- **用戶反饋**：👍/👎 評價題目質量
- **漸進式題庫**：靜態題庫 → AI 補充 → 混合優化

## 🎨 多題型設計方案

### 題型 1️⃣：猜動漫（Anime Quiz）

```
🎵 正在播放：《紅蓮の弓矢》
━━━━━━━━━━━━━━━━━━━━━━━
🎭 這是哪部動漫的主題曲？

[A. 進撃の巨人]  [B. 鬼滅の刃]
[C. 甲鉄城のカバネリ]  [D. 東京喰種]

⏱️ 15 秒
```

**適用場景**：
- 曲目元數據包含動漫標簽
- 藝術家是動漫歌手（LiSA、Aimer 等）
- 歌名包含日文片假名

**干擾選項策略**：
- 選擇同類型/同年代的動漫
- 選擇同一歌手演唱的其他動漫作品
- 選擇風格相似的動漫

---

### 題型 2️⃣：猜歌手（Artist Quiz）

```
🎵 正在播放
━━━━━━━━━━━━━━━━━━━━━━━
🎤 這首歌是誰演唱的？

[A. LiSA]  [B. Aimer]
[C. YOASOBI]  [D. 米津玄師]

⏱️ 15 秒
```

**適用場景**：
- 非動漫歌曲
- 知名流行歌手
- 題庫中有該歌手的其他作品

**干擾選項策略**：
- 選擇同類型歌手
- 選擇同時期活躍的歌手
- 選擇聲線相似的歌手

---

### 題型 3️⃣：猜歌名（Song Name Quiz）

```
🎵 正在播放
━━━━━━━━━━━━━━━━━━━━━━━
📝 這首歌的名字是？

[A. 紅蓮の弓矢]  [B. 紅蓮華]
[C. 炎]  [D. 心做し]

⏱️ 15 秒
```

**適用場景**：
- 默認題型（其他題型都不適用時）
- 歌名有特點的歌曲
- 經典熱門歌曲

**干擾選項策略**：
- 選擇同一歌手的其他歌曲
- 選擇相似歌名（如"紅蓮"系列）
- 選擇同專輯的其他曲目

## 🤖 AI 出題設計

### Claude API Prompt 模板

```
你是一位精通音樂的問答題生成專家。請為以下歌曲生成一道選擇題。

歌曲信息：
- 歌名：{track_name}
- 藝術家：{artist}
- 專輯：{album}（可選）
- Spotify URI：{spotify_uri}

題型：{quiz_type}
- anime：猜這是哪部動漫的主題曲
- artist：猜這首歌是誰演唱的
- song_name：猜這首歌的名字

要求：
1. 生成 1 個正確答案 + 3 個干擾選項
2. 干擾選項必須是真實存在的（動漫/歌手/歌名）
3. 干擾選項要有一定迷惑性，但不能太明顯
4. 難度適中，不要太簡單也不要太難
5. 如果無法確定答案，將 confidence 設為低于 0.7

請以 JSON 格式返回：
{
  "quiz_type": "anime",
  "question": "這是哪部動漫的主題曲？",
  "correct_answer": "進撃の巨人",
  "options": ["進撃の巨人", "鬼滅の刃", "甲鉄城のカバネリ", "東京喰種"],
  "difficulty": "easy",
  "explanation": "《紅蓮の弓矢》是《進撃の巨人》第一季的片頭曲",
  "confidence": 0.95,
  "tags": ["anime", "opening", "action"]
}
```

### AI 出題流程

```
用戶觸發 /quiz
  ↓
檢查題庫中是否有該曲目
  ├─ 有 → 直接使用（<1ms）
  └─ 無 → AI 生成
      ↓
    提取曲目元數據
      ↓
    判斷題型（anime/artist/song_name）
      ↓
    調用 Claude API（1-2s）
      ↓
    驗證 AI 返回的 confidence
      ├─ >= 0.7 → 使用並緩存
      └─ < 0.7 → 降級到默認題型
      ↓
    保存到 quiz_bank.json
      ↓
    顯示題目
```

## 📈 反饋優化策略

### 題目質量評分公式

```
quality_score = (likes - dislikes * 2) / max(total_plays, 1)

分級：
- quality_score >= 0.6  → 高質量（保留）✅
- 0.3 <= quality_score < 0.6  → 中等（觀察）⚠️
- quality_score < 0.3  → 低質量（待優化）❌
```

### 自動優化流程

```
定期任務（每周/每月）
  ↓
分析所有題目的反饋數據
  ↓
識別低質量題目（quality_score < 0.3）
  ↓
根據來源分類處理：
  ├─ manual（手動）→ 標記待人工修改
  └─ ai_generated（AI）→ 重新生成
      ↓
    保留原題目為 backup
      ↓
    調用 AI 重新生成
      ↓
    比較新舊版本
      ↓
    替換題庫
  ↓
生成優化報告
```

### 優化指標

| 指標 | 目標值 | 當前值 | 狀態 |
|------|--------|--------|------|
| 平均質量分數 | > 0.5 | - | 待測試 |
| 高質量題目占比 | > 70% | - | 待測試 |
| 用戶滿意度 | > 80% | - | 待測試 |
| AI 生成准確率 | > 85% | - | 待測試 |

## 📝 開發注意事項

### 性能優化
- 題庫查詢使用 HashMap（O(1) 時間復雜度）
- AI 調用使用異步處理，不阻塞主線程
- 緩存常用題目到內存
- 定期清理過期的游戲會話

### 錯誤處理
- API 調用失敗時的降級方案
- 題庫為空時的提示信息
- 無歌曲播放時的友好提示
- 超時處理

### 安全考慮
- 防止命令濫用（添加冷卻時間）
- 驗證用戶權限（在語音頻道中）
- 防止刷分行為
- 敏感信息不記錄（用戶隱私）

### 用戶體驗
- 響應時間 < 3 秒
- 按鈕點擊即時反饋
- 清晰的倒計時顯示
- 友好的錯誤提示
